What is HTML, really?

HTML stands for HyperText Markup Language. It is not a programming language; it is a markup language that tells the browser: “this is a heading”, “this is a paragraph”, “this is an image”, and so on.

When you open any website, your browser has downloaded an HTML file from a server using a protocol called HTTP and is drawing the page based on those tags.

Mental picture

Think of HTML as the skeleton of a page, CSS as the skin & clothing, and JavaScript as the brain & muscles that add behavior.

How the web works (super short)

  1. You type https://example.com in the browser.
  2. DNS turns that name into an IP address of a server.
  3. The browser sends an HTTP request: “give me your HTML”.
  4. The server responds with HTML (and then CSS/JS).
  5. Your browser parses it, builds a tree of nodes (DOM), and draws pixels on screen.

HTML Basics · Tags, Structure & Attributes

Learn the core building blocks: elements, attributes, comments, entities, and the skeleton of any HTML file.

Elements & tags

An HTML element usually has:

  • An opening tag – <p>
  • Content – Hello
  • A closing tag – </p>
Example: element
<p>This is a paragraph.</p>

Some elements are void (no closing tag): <br>, <img>, <hr>, etc.

The skeleton of every HTML5 page

Template you’ll reuse forever
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is my first HTML page.</p>
  </body>
</html>

The <head> is for metadata (page info, CSS, SEO) and is not directly visible. The <body> is where all visible content lives.

Attributes & global attributes

Attributes add extra information to elements in the form name="value".

Important global attributes (can be used on almost any element):

id – unique identifier
class – styling / grouping
style – inline CSS (avoid for big projects)
title – tooltip text
data-* – custom data for JS
Example with attributes
<p id="intro" class="lead" title="Hover tooltip">
  Hello with an id, class and title.
</p>

Comments & case insensitivity

Comments are ignored by the browser but useful for you: 

<!-- This will not be shown on the page -->

HTML tag names are not case-sensitive (<P> and <p> both work), but modern convention is all lowercase.

Entities · writing special characters

Because < and & are part of HTML syntax, we use entities:

Text, Headings, Links & Inline Elements

Make content readable and meaningful with headings, paragraphs, emphasis, and navigation links.

Headings hierarchy (h1–h6)

Headings create structure. Use them in order like an outline of a book:

  • <h1> – page main title (usually once).
  • <h2> – major sections.
  • <h3> – subsections, and so on.
<h1>HTML Course</h1>
<h2>Module 1: Basics</h2>
<h3>Lesson 1: Your first page</h3>

Paragraphs & line breaks

<p> wraps a block of text. <br> forces a new line inside text.

<p>This is one paragraph.</p>
<p>This is another paragraph.</p>

Address example:<br>
221B Baker Street<br>
London</code>

Emphasis, strong importance & other inline tags

Key inline tags:

  • <strong> – important text (screen readers emphasize it).
  • <em> – stress emphasis.
  • <b>, <i> – purely visual bold/italic.
  • <mark> – highlight.
  • <sub>, <sup> – subscript/superscript.
  • <span> – generic inline container.
<p>Use <strong>strong</strong> for important text.</p>
<p>Water is H<sub>2</sub>O and E = mc<sup>2</sup>.</p>
<p>This is <mark>highlighted</mark> text.</p>

Links (<a>) – how users move on the web

<a> creates a link. The most common attribute is href (destination).

  • Normal link: <a href="https://developer.mozilla.org/">MDN</a>
  • Email link: href="mailto:you@example.com"
  • Phone link: href="tel:+911234567890"
<a href="https://developer.mozilla.org/" target="_blank" 
   rel="noopener noreferrer">
  Learn from MDN
</a>

target="_blank" opens in a new tab. rel="noopener noreferrer" is a security best practice when using _blank.

Quotation & citation tags

HTML has tags just for quotes:

  • <q> – short inline quote.
  • <blockquote> – longer block quote.
  • <cite> – title of a work or citation source.
<p>Einstein said, <q>Imagination is more important than knowledge.</q></p>

<blockquote cite="https://developer.mozilla.org/">
  HTML is the standard markup language for Web pages.
</blockquote>

<p>Source: <cite>MDN Web Docs</cite></p>

Lists & Tables · Organizing Information

Use lists when order matters or doesn’t, and tables for real tabular data.

Unordered & ordered lists

Use <ul> (unordered list) for bullet points, and <ol> (ordered list) when sequence matters. 

<h3>Shopping list (ul)</h3>
<ul>
  <li>Milk</li>
  <li>Eggs</li>
  <li>Bread</li>
</ul>

<h3>Steps (ol)</h3>
<ol>
  <li>Boil water</li>
  <li>Add noodles</li>
  <li>Serve</li>
</ol>

Definition lists

A <dl> (definition list) is great for glossaries: 

<dl>
  <dt>HTML</dt>
  <dd>Structure of a web page.</dd>

  <dt>CSS</dt>
  <dd>Visual style of a web page.</dd>
</dl>

Tables – representing grid data

Tables are for tabular data, not general layout.

  • <table> – wrapper
  • <tr> – table row
  • <th> – header cell
  • <td> – data cell
  • <thead>, <tbody>, <tfoot> – grouping
<table>
  <thead>
    <tr>
      <th scope="col">Language</th>
      <th scope="col">Use</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Structure</td>
    </tr>
    <tr>
      <td>CSS</td>
      <td>Styling</td>
    </tr>
  </tbody>
</table>

scope="col" helps screen readers understand relation between headers and cells.

Images, Audio, Video & iframes

Make your pages rich with graphics and media while keeping them accessible and performant.

Images (<img>) and alt text

The HTML image tag is <img>. It is a void element (no closing tag).

  • src – path or URL of the image.
  • alt – text alternative (crucial for accessibility & SEO).
  • width / height – hint about dimensions.
  • loading="lazy" – loads image when near viewport.
<img src="https://via.placeholder.com/400x200"
     alt="Placeholder demo image"
     loading="lazy">

<figure> and <figcaption>

Wrap images and their captions in <figure>:

<figure>
  <img src="profile.jpg" alt="Photo of Pankaj">
  <figcaption>Pankaj – Web Developer</figcaption>
</figure>

Audio & video

<audio> and <video> allow controls to play media.

<audio controls>
  <source src="intro.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

<video controls width="320" poster="thumbnail.jpg">
  <source src="demo.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

iframes – embedding other pages

<iframe> lets you embed another webpage (maps, videos, etc.). 

<iframe src="https://example.com"
        title="Example site preview"
        width="400" height="200"
        loading="lazy">
</iframe>
Security Note: iframes can be abused. Many sites use extra headers like X-Frame-Options or CSP to control when they can be framed.

Forms & Inputs · Getting Data From Users

Forms are the heart of web apps: logins, signups, search, payments.

Form structure

The <form> element wraps inputs and buttons. Important attributes:

  • action – URL where data will be sent.
  • method – HTTP method (get or post).
<form action="/signup" method="post">
  ...
</form>

Labels & basic inputs

Always connect <label> with inputs using the for attribute (matches input’s id). 

<form>
  <label for="name">Name</label>
  <input id="name" name="name" type="text" required>

  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <button type="submit">Sign up</button>
</form>

Common input types

Useful type values for <input>:

TypePurpose
textSingle-line text
emailValidates email pattern
passwordHidden characters
numberNumeric input (+ step/min/max)
dateDate picker
checkboxOn/off toggle
radioChoose exactly one in a group
fileUploads a file
rangeSlider
colorColor picker
 
<label>
  <input type="checkbox" name="agree" required>
  I agree to the terms
</label>

<p>Choose role:</p>
<label><input type="radio" name="role" value="frontend"> Frontend</label>
<label><input type="radio" name="role" value="backend"> Backend</label>

Select, textarea & grouping

<label for="country">Country</label>
<select id="country" name="country">
  <option value="in">India</option>
  <option value="us">USA</option>
</select>

<label for="bio">Short bio</label>
<textarea id="bio" name="bio"></textarea>

<fieldset>
  <legend>Contact preference</legend>
  <label><input type="radio" name="contact" value="email"> Email</label>
  <label><input type="radio" name="contact" value="phone"> Phone</label>
</fieldset>

Validation attributes

HTML gives basic validation without JavaScript:

  • required – cannot be empty.
  • min, max, step – for numbers.
  • minlength, maxlength – for text.
  • pattern – custom regex pattern.
<input type="password" required minlength="8"
       placeholder="At least 8 characters">
Client-side validation is helpful, but you must always validate again on the server in real applications.

Semantic Layout · Building Real Page Structure

Use dedicated tags so your HTML is understandable to browsers, screen readers, and search engines.

Why semantics matter

Semantic HTML means using tags that describe their meaning: <header> for page headers, <nav> for navigation, <main> for main content, etc.

Benefits: better accessibility, SEO, and easier collaboration.

Common layout tags

<header>
<nav>
<main>
<section>
<article>
<aside>
<footer>
<header>
  <h1>My Blog</h1>
  <nav>
    <a href="#articles">Articles</a>
    <a href="#about">About</a>
  </nav>
</header>

<main>
  <section id="articles">
    <article>
      <h2>First Post</h2>
      <p>Post content...</p>
    </article>
  </section>
  <aside>Sidebar, related links, ads</aside>
</main>

<footer>© 2025 My Blog</footer>

Other semantic helpers

  • <time> – represents a date or time.
  • <address> – contact details.
  • <main> – only one per page, central content.
<article>
  <h2>Meetup</h2>
  <p>Happening on 
    <time datetime="2025-12-05">5 December 2025</time>.
  </p>
  <address>
    Organized by Dev Club, NIT Jalandhar
  </address>
</article>

Head, Meta, SEO & Performance Basics

Make your page discoverable, shareable, and mobile-friendly.

The <head> section

The <head> contains information about the page: title, meta tags, CSS/JS links, icons, etc. 

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Portfolio</title>
  <meta name="description" content="Portfolio of Pankaj, web developer.">
  <link rel="icon" href="favicon.ico">
</head>

SEO essentials

  • Unique, descriptive <title> for each page.
  • Helpful meta name="description" (up to ~160 characters).
  • Proper heading structure (h1, h2, ...).
  • Meaningful URLs (done on backend, but HTML links should be clear).
  • alt text on images.

Social sharing (Open Graph basics)

When someone shares your link,

Open Graph tags control how it looks:

  • og:title – title of the page.
  • og:description – brief summary.
  • og:image – thumbnail image URL.
  • og:type – type of content (e.g., website, article).
<meta property="og:title" content="Pankaj's Portfolio">
<meta property="og:description" content="Web Developer from NIT Jalandhar">
<meta property="og:image" content="thumbnail.jpg">
<meta property="og:type" content="website">

Performance basics

  • Use loading="lazy" for images/iframes.
  • Compress images (WebP preferred).
  • Keep CSS and JS small.
  • Use semantic HTML to help browsers render faster.
  • Minimize unnecessary fonts/scripts.

Accessibility & Best Practices

Build websites usable by everyone—including screen readers, keyboard-only users, and people with disabilities.

Accessibility fundamentals

  • Always use alt text for images.
  • Use proper heading order (no skipping h2 → h4).
  • Forms must always have labels.
  • Links must make sense (avoid “click here”).
  • Color contrast must be readable.

Keyboard navigation

Users must be able to use your website with only the keyboard.

  • Tab should move between interactive items.
  • Use :focus styles so users know where they are.
a:focus, button:focus {
  outline: 2px solid #38bdf8;
  border-radius: 6px;
}

ARIA attributes

ARIA helps screen readers understand complex components.

  • role="navigation"
  • aria-label="Main menu"
  • aria-expanded="false"

SEO + Accessibility overlap

  • Good headings → better screen readers & SEO
  • Alt text → accessibility + Google images ranking
  • Semantic tags → better crawling + more readable
